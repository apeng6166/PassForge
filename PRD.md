## 1. 产品概述

### 1.1 背景

用户在注册账号、重置密码或管理多平台凭证时，常需要强密码。本地工具或浏览器插件虽可用，但一个**轻量、无需安装、隐私友好**的 Web 密码生成器仍有明确需求。

### 1.2 产品定位

**PassForge** 是一款在浏览器中运行的密码生成器网站，帮助用户快速生成符合安全策略的随机密码，并提供强度评估与一键复制等能力。

### 1.3 产品愿景

成为「打开即用、安全可信、体验简洁」的在线密码生成工具。

### 1.4 目标用户

| 用户类型 | 描述 | 核心诉求 |
|----------|------|----------|
| 普通互联网用户 | 注册新账号、忘记密码需重置 | 快速生成强密码、操作简单 |
| 开发者 / IT 人员 | 为测试环境、API Key 等生成凭证 | 可配置长度与字符集、批量生成 |
| 安全意识较强用户 | 关注隐私与本地处理 | 密码不在服务端存储、透明安全说明 |

## 2. 产品目标

### 2.1 业务目标

- 提供稳定可用的密码生成核心能力
- 建立用户对「本地生成、不上传密码」的信任
- 为后续扩展（密码强度检测、 passphrase、历史记录等）预留架构空间

### 2.2 成功指标（MVP 阶段）

| 指标 | 目标 |
|------|------|
| 首次生成密码耗时 | ≤ 3 秒（含打开页面） |
| 核心功能可用率 | ≥ 99% |
| 移动端可用性 | 主流手机浏览器正常使用 |
| 用户完成「生成 → 复制」转化率 | 待上线后统计 |

### 2.3 非目标（MVP 不做）

- 用户账号体系与云端密码库
- 密码同步、跨设备共享
- 浏览器插件 / 桌面客户端
- 付费订阅与商业化

## 3. 用户场景

### 场景 1：注册新账号

> 用户在某网站注册，需要 16 位含大小写、数字、符号的密码。

**流程**：打开 PassForge → 调整长度与字符类型 → 生成 → 复制 → 粘贴到注册页。

### 场景 2：批量生成临时密码

> 开发者需要为 5 个测试账号生成不同密码。

**流程**：设置参数 → 选择生成数量 → 批量生成 → 逐条或整体复制。

### 场景 3：移动端快速使用

> 用户在手机上注册 App，需要立即生成密码。

**流程**：手机浏览器访问 → 一键生成默认强密码 → 复制。

## 4. 功能需求

### 4.1 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 随机密码生成 | 核心能力，MVP 必须 |
| P0 | 长度与字符集配置 | 用户可自定义规则 |
| P0 | 一键复制 | 降低使用摩擦 |
| P0 | 密码强度指示 | 视觉化反馈 |
| P1 | 批量生成 | 一次生成多条 |
| P1 | 排除易混淆字符 | 如 `0/O`、`1/l/I` |
| P1 | 深色模式 | 跟随系统或手动切换 |
| P2 | Passphrase（词组密码） | 基于单词列表 |
| P2 | 多语言界面 | 中/英等 |
| P2 | PWA / 离线可用 | 弱网或无网场景 |

### 4.2 功能详细说明

#### F1：密码生成（P0）

**描述**：根据用户配置，在客户端生成加密学安全的随机密码。

**规则**：

- 默认长度：16
- 可调范围：4–128（可讨论上限）
- 字符类型（多选）：
  - 大写字母 `A–Z`
  - 小写字母 `a–z`
  - 数字 `0–9`
  - 特殊符号（可配置符号集）
- 至少选择一种字符类型，否则禁止生成并提示
- 使用 `crypto.getRandomValues()` 或等价 CSPRNG，**禁止**使用 `Math.random()`

**交互**：

- 「生成」按钮：按当前配置生成新密码
- 「重新生成」：同配置再次随机
- 生成后自动更新强度指示与预览区

**验收标准**：

- [ ] 相同配置多次生成结果不同（概率意义上）
- [ ] 生成密码 100% 符合所选字符集与长度
- [ ] 未选任何字符类型时无法生成并有明确提示

---

#### F2：密码展示与复制（P0）

**描述**：清晰展示生成结果，支持一键复制到剪贴板。

**规则**：

- 等宽字体展示密码
- 支持「复制」按钮；复制成功后 Toast 提示
- 可选：复制后 N 秒自动清空展示（增强安全，P1）
- 不在 URL、LocalStorage、服务端日志中持久化密码

**验收标准**：

- [ ] 复制功能在 Chrome / Firefox / Safari / Edge 可用
- [ ] 移动端复制可用（含 iOS Safari 降级方案）

---

#### F3：密码强度评估（P0）

**描述**：对当前密码给出强度等级与简要说明。

**规则**：

- 等级建议：弱 / 中 / 强 / 极强（或 1–4 级）
- 评估维度：长度、字符种类、熵值估算
- 仅作参考，文案避免绝对化（如「不可破解」）

**验收标准**：

- [ ] 长度与字符种类变化时强度实时更新
- [ ] 强度颜色与文案一致（如红/黄/绿）

---

#### F4：生成参数配置（P0）

**描述**：用户可调整密码生成规则。

**配置项**：

| 配置项 | 类型 | 默认值 |
|--------|------|--------|
| 密码长度 | 滑块 + 数字输入 | 16 |
| 大写字母 | 开关 | 开 |
| 小写字母 | 开关 | 开 |
| 数字 | 开关 | 开 |
| 特殊符号 | 开关 | 开 |
| 排除易混淆字符 | 开关 | 关（P1） |

**验收标准**：

- [ ] 配置变更后，下次生成生效
- [ ] 参数在会话内可保持（可选：LocalStorage 仅存配置，不存密码）

---

#### F5：批量生成（P1）

**描述**：一次生成多条不重复密码。

**规则**：

- 数量范围：1–50（可配置上限）
- 列表展示，支持单条复制或「复制全部」
- 批量结果不自动下载文件（MVP 可不做导出）

---

#### F6：Passphrase 模式（P2）

**描述**：使用随机单词组合生成易记但足够长的密码，如 `correct-horse-battery-staple`。

**规则**：

- 单词数量：3–8
- 分隔符：`-` / `_` / `.` / 无
- 可选首字母大写、随机数字后缀
